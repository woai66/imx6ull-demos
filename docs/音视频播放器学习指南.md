# 嵌入式 Linux 音视频播放器学习指南

> 本指南面向有一定嵌入式基础、想快速上手 Linux 音视频开发的开发者。
> 新手可通读了解全貌，再按需补基础。

---

## 播放器系统架构
   
```
┌─────────────────────────────────────────────┐
│  应用层 (Application)                        │
│  ├── GUI 界面 (Qt/LVGL)                     │
│  ├── 播放控制 (播放/暂停/进度/音量)          │
│  ├── 播放列表管理                            │
│  └── 网络流媒体 (HTTP/RTP/RTSP)              │
├─────────────────────────────────────────────┤
│  多媒体框架层 (Multimedia Framework)         │
│  ├── GStreamer Pipeline                     │
│  │   └── 元件: filesrc → demux → decode → sink │
│  └── FFmpeg (libavformat + libavcodec)      │
├─────────────────────────────────────────────┤
│  编解码层 (Codec)                            │
│  ├── 音频解码: MP3, AAC, WAV                │
│  ├── 视频解码: H.264, MPEG-4, MJPEG         │
│  └── 硬件加速: VPU (i.MX6 VPU 支持 H.264)   │
├─────────────────────────────────────────────┤
│  设备接口层 (Device Interface)               │
│  ├── 视频输出: Framebuffer / DRM / V4L2     │
│  ├── 音频输出: ALSA (libasound)             │
│  └── 输入设备: 触摸屏 / 按键 / 红外          │
├─────────────────────────────────────────────┤
│  内核驱动层 (Kernel Driver)                  │
│  ├── LCD 驱动 (MIPI/RGB/HDMI)               │
│  ├── 音频驱动 (SAI/SSI → WM8960/ES8328)     │
│  └── GPU/VPU 驱动 (vivante gc400)           │
├─────────────────────────────────────────────┤
│  硬件层 (Hardware)                           │
│  ├── CPU: i.MX6ULL (Cortex-A7)              │
│  ├── RAM: 512MB DDR3                        │
│  ├── 存储: eMMC/SD                          │
│  ├── 显示: RGB LCD / HDMI                   │
│  └── 音频: I2S Codec                        │
└─────────────────────────────────────────────┘
```

## 前置知识清单

| 知识点 | 掌握程度 | 说明 |

| Makefile / CMake 构建 | 必须 | 交叉编译必备 |

### 2. Linux 驱动基础
| 知识点 | 掌握程度 | 说明 |

| Platform 总线 / 设备树 | 了解 | 理解设备如何被内核识别 |
| Framebuffer 子系统 | 必须 | LCD 显示输出 |
| ALSA 音频子系统 | 必须 | 音频播放接口 |

## 分阶段学习路线

### 第一阶段：Linux 环境熟悉（2-3 周）
**目标**: 能在开发板上跑起来自己的 C 程序

1. **交叉编译环境搭建**
   - 安装 Ubuntu 虚拟机
   - 配置 `arm-linux-gnueabihf-gcc` 工具链
   - 写一个 `hello.c`，交叉编译，放到开发板运行
   - 学会使用 NFS 挂载调试程序

2. **Linux 基础编程**
   - 文件 I/O：`open()` 一个 MP3 文件，用 `read()` 读取前 1024 字节
   - `mmap()`：理解内存映射，后续用于 Framebuffer 直接写屏
   - `ioctl()`：理解设备控制，所有音视频设备都用它配置

3. **Makefile 与 CMake**
   - 学会写多文件项目的 Makefile
   - 学会交叉编译第三方库（如 libalsa、libmad）

### 第二阶段：音频播放打通（2-3 周）
**目标**: 播放一段 WAV 音频文件

1. **ALSA 基础**
   - 学会 `aplay` 命令播放测试音频
   - 用 ALSA API 打开 PCM 设备（`/dev/snd/pcmC0D0p`）
   - 配置参数：采样率 44100、16bit、立体声
   - 循环写入 PCM 数据，听到声音

2. **WAV 文件解析**
   - 理解 WAV 文件头结构（RIFF chunk、fmt chunk、data chunk）
   - 写代码解析 WAV 头，提取 PCM 数据
   - 把提取的 PCM 数据送给 ALSA 播放

3. **MP3 解码（软解）**
   - 移植 `libmad` 或 `minimp3` 解码库
   - 读取 MP3 文件 → 解码为 PCM → 送给 ALSA
   - 理解：MP3 是压缩格式，必须先解码成 PCM 才能播放

### 第三阶段：视频显示打通（2-3 周）
**目标**: 在 LCD 上显示一张 BMP 图片

1. **Framebuffer 基础**
   - 打开 `/dev/fb0`
   - 用 `ioctl(FBIOGET_VSCREENINFO)` 获取屏幕参数（分辨率、色深）
   - 用 `mmap()` 映射显存
   - 直接往显存写 RGB 数据，显示纯色或条纹

2. **BMP 图片显示**
   - 理解 BMP 文件格式（文件头 + DIB 头 + 像素数据）
   - 解析 BMP，将 RGB 数据写入 Framebuffer
   - 处理不同色深（16bit/24bit/32bit）和行对齐

3. **YUV 转 RGB**
   - 视频解码器通常输出 YUV420 格式
   - 理解 YUV → RGB 转换公式
   - 写转换函数，显示 YUV 测试图

### 第四阶段：视频播放打通（3-4 周）
**目标**: 播放一段无音频的视频（如 MJPEG）

1. **AVI 文件解析**
   - 理解 AVI 容器结构（RIFF → LIST → hdrl/movi）
   - 解析视频流数据（MJPEG 帧）
   - 理解时间戳和帧率控制

2. **MJPEG 解码与显示**
   - 移植 `libjpeg-turbo` 进行 JPEG 解码
   - 每帧解码为 RGB/YUV → 写入 Framebuffer
   - 加入帧率控制（`usleep()` 或更精确的定时器）

3. **音视频同步基础**
   - 理解 PTS/DTS 概念
   - 简单同步：视频按音频时钟对齐
   - 双线程：一个线程解码音频，一个线程解码视频

### 第五阶段：引入多媒体框架（3-4 周）
**目标**: 用 GStreamer 播放 MP4 视频

1. **GStreamer 基础**
   - 理解 Pipeline、Element、Pad、Bus 概念
   - 命令行测试：`gst-launch-1.0 playbin uri=file:///test.mp4`
   - 用 C API 构建 Pipeline：`gst_parse_launch()`

2. **交叉编译 GStreamer**
   - 编译 `gstreamer` + `gst-plugins-base` + `gst-plugins-good`
   - 可能需要编译 `gst-libav` 或 `gst-imx`（i.MX6 硬件加速）

3. **自定义 Pipeline**
   - `filesrc → qtdemux → h264parse → imxvpudec → imxipuvideosink`
   - 提取视频帧进行 OSD 叠加（时间戳、进度条）

4. **可选：FFmpeg 路线**
   - 如果 GStreamer 编译困难，可用 FFmpeg 替代
   - `avformat_open_input → avcodec_decode → sws_scale → fb_write`

### 第六阶段：GUI 与播放器完善（3-4 周）
**目标**: 一个带界面的完整播放器

1. **GUI 框架选择**
   - **Qt**: 功能强，支持 GPU 加速，但体积大（>50MB）
   - **LVGL**: 轻量（<1MB），适合嵌入式，推荐
   - **直接 Framebuffer**: 自己画按钮和进度条，最轻量

2. **播放器功能**
   - 播放/暂停/停止控制
   - 进度条拖动（seek）
   - 音量调节
   - 文件列表浏览
   - 循环/随机播放模式

3. **优化与问题排查**
   - 内存占用优化（i.MX6ULL 只有 512MB）
   - 卡顿问题：解码线程优先级、双缓冲
   - 音画不同步：音视频时钟同步策略

## 核心技术详解

### ALSA 音频播放核心流程

```c
// 1. 打开 PCM 设备
snd_pcm_open(&pcm, "default", SND_PCM_STREAM_PLAYBACK, 0);

// 2. 配置参数
snd_pcm_set_params(pcm, SND_PCM_FORMAT_S16_LE,
                   SND_PCM_ACCESS_RW_INTERLEAVED,
                   2,          // 声道数
                   44100,      // 采样率
                   1,          // 软重采样
                   500000);    // 延迟 (us)

// 3. 循环写入数据
snd_pcm_writei(pcm, buffer, frames);

// 4. 关闭
snd_pcm_drain(pcm);
snd_pcm_close(pcm);
```

### Framebuffer 显示核心流程

```c
// 1. 打开 framebuffer
int fb = open("/dev/fb0", O_RDWR);

// 2. 获取屏幕信息
struct fb_var_screeninfo vinfo;
ioctl(fb, FBIOGET_VSCREENINFO, &vinfo);
int width = vinfo.xres;
int height = vinfo.yres;
int bpp = vinfo.bits_per_pixel;  // 通常为 32 (ARGB) 或 16 (RGB565)

// 3. 映射显存
size_t screensize = width * height * bpp / 8;
uint8_t *fbmem = mmap(NULL, screensize, PROT_READ|PROT_WRITE, MAP_SHARED, fb, 0);

// 4. 直接写像素 (以 32bpp 为例)
for (int y = 0; y < height; y++) {
    for (int x = 0; x < width; x++) {
        uint32_t *pixel = (uint32_t *)(fbmem + (y * width + x) * 4);
        *pixel = 0xFF0000FF;  // 蓝色 (ABGR)
    }
}

// 5. 清理
munmap(fbmem, screensize);
close(fb);
```

### GStreamer 播放 Pipeline 示例

```c
// 最简单的播放 pipeline
GstElement *pipeline = gst_parse_launch(
    "playbin uri=file:///mnt/test.mp4",
    NULL);
gst_element_set_state(pipeline, GST_STATE_PLAYING);

// 自定义 pipeline（使用 i.MX6 VPU 硬件解码）
GstElement *pipeline = gst_parse_launch(
    "filesrc location=/mnt/test.mp4 ! "
    "qtdemux name=d "
    "d.video_0 ! queue ! h264parse ! imxvpudec ! imxipuvideosink "
    "d.audio_0 ! queue ! decodebin ! alsasink",
    NULL);
```

| 库名 | 用途 | 交叉编译难度 |
|------|------|-------------|

## i.MX6ULL 硬件加速说明

i.MX6ULL **没有 VPU**（注意：i.MX6Q/i.MX6D 才有 VPU），这意味着：

- **H.264 硬解码**: ❌ 不支持（i.MX6ULL 没有 VPU）
- **GPU 2D 加速 (PXP)**: ✅ 支持，可做颜色空间转换、缩放
- **音频解码**: ✅ 软件解码即可，MP3/AAC 软解 CPU 占用低
- **视频播放建议**:
  - 分辨率 ≤ 480p (854x480)
  - 格式：MJPEG（每帧独立 JPEG，易解码）或 MPEG-4 Part 2
  - 帧率：≤ 30fps
  - 避免 H.264 1080p，CPU 带不动

如果是 i.MX6Q/6D，则可以通过 `imxvpudec` GStreamer 插件硬解 H.264。

## 播放器项目拆解（MVP 路线）

如果直接做完整播放器太复杂，可以按这个顺序实现 **最小可用播放器**：

1. **Week 1-2: 纯 ALSA 播放器**
   - 只支持 WAV 格式
   - 命令行运行：`./playwav music.wav`
   - 代码量：~200 行

2. **Week 3-4: 加入 MP3 支持**
   - 集成 libmad 解码 MP3
   - 代码量：~500 行

3. **Week 5-6: 加入 Framebuffer 显示**
   - 显示 BMP 图片
   - 代码量：+300 行

4. **Week 7-8: MJPEG 视频播放**
   - 解析 AVI，解码 JPEG，显示到 LCD
   - 代码量：+500 行

5. **Week 9-10: 引入 GStreamer**
   - 用 `playbin` 播放 MP4
   - 加入简单控制（播放/暂停）
   - 代码量：+300 行

6. **Week 11-12: GUI 与完善**
   - 加入 LVGL 界面
   - 文件列表、进度条、音量
   - 代码量：+1000 行

**总计**: 约 3000 行代码，一个可用的嵌入式播放器。
